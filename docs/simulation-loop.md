# 시뮬레이션 루프 및 행동 설계

## 1. 기본 시간 구조
- 게임은 틱(tick) 단위로 진행한다.
- 한 틱마다 모든 활성 NPC와 필요한 오브젝트를 갱신한다.
- 화면 배속은 틱 간격을 줄이는 방식으로 처리한다.

## 2. NPC 업데이트 순서
각 NPC는 아래 순서로 처리한다.
1. 욕구 증가
2. 주변 감지
3. 행동 후보 생성
4. 점수 계산
5. 행동 선택
6. 실행
7. 결과 반영
8. 기억/관계 갱신
9. 로그 생성

## 3. 행동 후보
MVP 기준 행동 후보:
- Idle
- Move
- Eat
- Sleep
- GatherGrass
- ChopTree
- MineRock
- CarryResource
- GoHome
- Talk
- Flee
- Attack

## 4. 행동 선택 방식
유틸리티 AI 기반으로 시작한다.

예시:
```text
EatScore = Hunger * 0.9 + NearbyFoodBonus * 0.5
SleepScore = Fatigue * 0.8 + NightBonus * 0.4
FleeScore = Fear * 0.9 + ThreatLevel * 0.8 - Courage * 0.5
ChopTreeScore = WoodNeed * 0.7 + JobAffinity * 0.6
```

가장 점수가 높은 행동을 선택한다.

## 5. 이동 구조
- 내부는 Grid 이동
- 한 틱당 1칸 이동 또는 이동 진행도 누적
- 충돌 방지를 위해 `판단 단계 -> 적용 단계`로 분리

## 6. 오브젝트 상호작용
### 6.1 나무
- 접근
- 벌목 가능 여부 확인
- 벌목 진행
- 목재 드롭 또는 인벤토리 획득

### 6.2 풀
- 채집 가능 여부 확인
- 식량 확보
- 재생성 타이머 시작

### 6.3 바위
- 채굴 가능 여부 확인
- 돌 획득

### 6.4 집
- 귀가
- 수면
- 거주 상태 반영

## 7. 이벤트 로그
다음 사건은 로그에 남긴다.
- 자원 채집 시작/완료
- 싸움 발생
- 도망
- 부상
- 사망
- 건설 시작/완성
- 관계 변화

## 8. 성능 방안
- 화면 내부 NPC: 고해상도 업데이트
- 근처 구역 NPC: 중간 해상도 업데이트
- 먼 NPC: 요약 상태 업데이트

## 9. 종료 또는 상태 변화 기준
- 모든 NPC는 사망, 수면, 작업, 공포, 전투 상태를 가질 수 있다.
- 마을 상태는 식량 부족, 자원 과잉, 안정, 불안, 붕괴 위험 등으로 요약 가능하다.
