# MVP 개발 로드맵

## 1단계: 화면과 월드 기초
목표: 쿼터뷰 월드가 보이고 오브젝트를 개별 선택할 수 있어야 한다.
- 쿼터뷰 타일맵 구성
- 나무, 풀, 바위, 집 프리팹 배치
- 오브젝트 선택 시 정보 표시
- Sorting Order 자동 처리

## 2단계: NPC 기초
목표: 스피키 A/B/C/D가 생성되고 이동해야 한다.
- 스피키 A/B/C/D 생성
- 성격 프리셋 랜덤 배정
- Grid 이동
- Idle/Move 상태 표시

## 3단계: 생존 루프
목표: NPC가 스스로 먹고 쉬고 귀가해야 한다.
- 배고픔 증가
- 피로 증가
- 식사 행동
- 귀가 및 수면

## 4단계: 자원 루프
목표: NPC가 오브젝트와 상호작용해 자원을 모아야 한다.
- 풀 채집
- 나무 벌목
- 바위 채굴
- 인벤토리 반영
- 저장고 입고

## 5단계: 기본 로그
목표: 플레이어가 변화 원인을 볼 수 있어야 한다.
- 사건 로그 패널
- NPC 현재 행동 표시
- 오브젝트 상태 표시

## 6단계: 건설 기초
목표: 마을이 실제로 확장되기 시작해야 한다.
- 집 건설 예약
- 자원 운반
- 건설 진행도
- 완성 후 거주 가능

## MVP 완료 기준
- 쿼터뷰 맵에서 입체적으로 보이는 오브젝트가 존재한다.
- 집, 나무, 풀, 바위가 각각 개별 오브젝트로 선택된다.
- 스피키 A/B/C/D가 각자 랜덤 성격으로 생성된다.
- NPC가 욕구와 자원 상황에 따라 행동을 바꾼다.
- 자원 수집 및 저장이 이루어진다.
- 플레이어가 로그와 상세 패널로 상황을 관찰할 수 있다.
