# 전체 아키텍처 설계

## 1. 프로젝트 정의
플레이어는 쿼터뷰 마을을 관찰하며, 개별 오브젝트와 개별 NPC들이 자원 수집, 생존, 건설, 관계 형성, 갈등을 통해 마을이 발전하거나 붕괴하는 과정을 지켜본다.

## 2. 설계 원칙
- 표현과 로직을 분리한다.
- 오브젝트는 주요 상호작용 대상마다 개별 인스턴스로 존재한다.
- 모든 NPC는 동일 규칙으로 동작한다.
- 쿼터뷰는 표현 방식이고, 내부 시뮬레이션은 2D 그리드 좌표와 상태 머신 기반으로 운영한다.

## 3. 주요 서브시스템
### 3.1 World System
- 지형 타일 관리
- 개별 오브젝트 등록/삭제
- 타일 점유 상태 관리
- 좌표 변환(Grid -> World)

### 3.2 NPC System
- 욕구(배고픔, 피로, 안전)
- 감정
- 관계
- 기억
- 현재 상태
- 행동 후보 선택

### 3.3 Object System
- 나무, 풀, 바위, 집, 창고, 농장, 울타리 등
- 위치, 내구도, 자원량, 점유 상태, 상호작용 가능 여부 보유

### 3.4 Simulation System
- 틱 단위 시간 진행
- NPC 업데이트
- 자원 생산/소모
- 건설 진행
- 사건 처리

### 3.5 UI System
- 시간, 배속, 사건 로그
- NPC 상세 패널
- 오브젝트 상세 패널
- 마을 자원 패널

## 4. 레이어 구조
- Ground Layer: 흙, 길, 물, 농지
- Object Layer: 나무, 바위, 풀, 건물
- NPC Layer: 주민, 동물, 적대 개체
- Event Layer: 시체, 드롭 아이템, 화재, 흔적

## 5. 스피키 규칙
- 스피키 A/B/C/D는 일반 NPC다.
- 이름만 고정이다.
- 성격 프리셋은 시작 시 랜덤 배정한다.
- 스피키만의 예외 규칙은 두지 않는다.

## 6. 기술 방향
- Unity 2D
- 쿼터뷰 2.5D 시각 표현
- Grid 기반 내부 좌표
- Prefab 기반 개별 오브젝트 관리

## 7. 확장 우선순위
1. MVP: 이동, 자원 채집, 귀가, 로그
2. 중기: 건설, 저장, 역할 분담
3. 후기: 관계, 갈등, 죽음, 사건 체인
4. 장기: 종교, 법, 문화, 사회 구조
